Influence Maximization in Python - CELF vs. RIS

The influence maximization (IM) problem seeks a set of seed nodes in a given network to maximize the expected number of nodes activated via an influence cascade initiated at that seed set. A previous post compared two IM algorithms - the Greedy algorithm of Kempe et al. (2003) and the CELF algorithm of Leskovec et al. (2007). For years, CELF (and its modified CELF++ version by Goyal et al. 2011) represented the fastest theoretically guaranteed IM algorithms, and the subsequent literature focussed largely on improving computational efficiency through the use of heuristics that sacrificed theoretical guarantees in favor of speed. More recently, however, a new approach to the problem - known as Reverse Influence Sampling (RIS) - has emerged which is both fast and theoretically guaranteed, and is therefore generally accepted to represent the state-of-the-art in IM methods. This post walks through an implementation of the RIS algorithm in Python and compare its solutions and computational speed to the CELF algorithm from the previous post.

The Reverse Influence Sampling Algorithm¶

The key insight enabling the RIS family of algorithms is the concept of a random reverse reachable set introduced to the IM problem by Borgs et al. (2014). A reverse reachable (RR) set for an arbitrary node $v$ is generated by first sampling a graph $g$ from the distribution generated by removing each edge $e$ according to its propagation probability $1-p_e$ and then taking the set of nodes in $g$ that can "reach" $v$ (in this post we'll assume that all edges share the same propagation probability so that $p_e=p$). A random reverse reachable (RRR) set is a RR set for a node selected uniformly at random.

Intuitively, if a node $u$ appears in a RR set of another node $v$, then there is a directed path from $u$ to $v$ and a diffusion process from a seed set containing $u$ therefore has some probability of activating $v$. This connection between RR sets and node activations is formalized in a lemma, which states that the probability that a diffusion process from any seed set $S$ will activate any node $v$ is equal to the probability that $S$ overlaps an RR set for $v$ (at least one node in $S$ is contained within a RR set for $v$).

Based on this lemma, the family of RIS algorithms proceed in two steps:

  1. Generate a set $R$ of mc independent RRR sets. This is implemented by traversing all paths from $v$ in the transpose of the input graph $g^T$, which is generated by reversing the direction of all edges in g.

  2. Solve the "maximum coverage problem" of selecting $k$ nodes to cover the maximum number of RRR sets in R using the standard greedy algorithm, which obtains a $(1-1/e-\epsilon)$-approximate solution $S_k^*$ to the problem. In practice, the seed set is constructed by iteratively choosing the node that appears in the most RRR sets and upon the selection of each node, the RRR sets featuring that node are removed.

This approach works because for any seed set $S$, the fraction of RRR sets in $R$ covered by $S$ is an unbiased estimator of the spread of $S$ (due to the lemma described above). Therefore, a seed set that covers a large number of RR sets in $R$ is likely to have a large expected influence, which makes it a good solution to the IM problem.

The key to the very high computational performance of the RIS algorithm is that, unlike Greedy or CELF, it doesn't repeat the spread computation procedure to incrementally construct a solution. Instead, it performs all the Monte Carlo simulations/sampling up front to construct $R$, from which it then selects the entire seed set. The method therefore avoids the "wasted" spread computations of the greedy algorithm and most of its successors because the generated RRR sets are used to inform the spread of all nodes within the network.

Below, I describe the implementation of the RIS algorithm using two separate functions. The first, get_RRS(), takes a graph and propagation probability and spits out a random reverse reachable set. The second, ris(), uses the first function to generate a large number of random reverse reachable sets, which are then used to select the influential seed set. We start by loading a few modules.

Create Random Reverse Reachable Set¶

The get_RRS() function below takes a network object and produces a RRR set from that network. The network is defined within a dataframe G, where each row represents a directed edge in the network, and the two columns ['source','target'] describe the source node and target node of a given edge, respectively. There are many ways to represent a network in Python (including the popular igraph and NetworkX packages) but this setup helps with transparency and can be used to conveniently take the transpose of a graph by simply swapping the first two columns. The function proceeds in three steps:

  1. Randomly select the source node from the set of all potential source nodes contained in the source column of G.

  2. Sample an instance g from the network G by creating an existence probability for each edge in G based on the IC parameter p and the weight of that particular edge, which can be seen as the number of connections that scale the general independent cascade propagation probability p. Compare these probabilities with a uniform random draw to simulate whether that edge is sampled, and extract those edges that remain into a new dataframe, g, which represents the sampled graph. This is a very similar method to that used in the IC() function to simulate a propagation and indeed can be shown to be equivalent.

  3. Produce the RR set itself through an iterative procedure that first finds all nodes with edges leading into source, which are then added to the RRS list object. The loop then finds all the nodes with edges leading into the new nodes found in the previous step and so on. As mentioned, the dataframe representation of the network makes this process very simple because we do not need to create the transpose graph $g^T$.

def get_RRS(G,p):   
    """
    Inputs: G:  Ex2 dataframe of directed edges.
                Columns are ['source','target']. Each row is an edge.
            p:  Disease propagation probability
    Return: A random reverse reachable set expressed as a list of nodes
    """
    
    # Step 1. Select random source node
    source = random.choice(list(set(G['source'].tolist())))
    
    # Step 2. Get an instance of g from G by sampling edges  
    g = G.copy()
    g['success'] = np.random.uniform(0,1,g.shape[0]) < p
    g = g.loc[g['success']]

    # Step 3. Construct reverse reachable set of the random source node
    new_nodes, RRS0 = [source], [source]   
    while new_nodes:
        
        # Limit to edges that flow into the source node
        temp = g.loc[g['target'].isin(new_nodes)]

        # Extract the nodes flowing into the source node
        temp = temp['source'].tolist()

        # Add new set of in-neighbors to the RRS
        RRS = list(set(RRS0 + temp))

        # Find what new nodes were added
        new_nodes = list(set(RRS) - set(RRS0))

        # Reset loop variables
        RRS0 = RRS[:]

    return(RRS)

Create RIS algorithm¶

The rrs() function implements the actual RIS procedure and proceeds in two steps:

  1. Generate a large collection of random reverse reachable sets, R, by looping over the get_rrs() function.

  2. Run the "maximum greedy coverage" algorithm which simply finds the node that appears in the most RRR sets. Once this is found, those RRR sets are removed from the collection R, and the process is repeated until k nodes are chosen and stored in the list object SEED.

Along the way we also keep track of the time, which we use to compare with the CELF running time below.

def ris(df,k,p=0.5,mc=1000):    
    """
    Inputs: df: Ex2 dataframe of directed edges.
                Columns are ['source','target']. Each row is an edge
            k:  size of seed set
            p:  Disease propagation probability
    Return: A seed set of nodes as an approximate solution to the IM problem
    """
    
    # Step 1. Generate the collection of random RRSs
    R, start_time = [], time.time()
    for _ in range(mc):
        R.append(get_RRS(df,p))

    # Step 2. Choose nodes that appear most often (maximum coverage greedy algorithm)
    SEED, timelapse = [], []
    for _ in range(k):
        
        # Find node that occurs most often in R
        flat_list = [item for sublist in R for item in sublist]
        seed = Counter(flat_list).most_common()[0][0]
        
        # Remove RRSs containing last chosen seed 
        R = [item for item in R if seed not in item]
        
        # Add to outputs
        SEED.append(seed)
        timelapse.append(time.time() - start_time)
    
    return(sorted(SEED),timelapse)

def IC(df,S,p=0.5,mc=1000):  
    """
    Input:  dataframe representation of network, set of seed nodes, propagation probability
            and the number of Monte-Carlo simulations
    Output: average number of nodes influenced by the seed nodes
    """
    
    # Loop over the Monte-Carlo Simulations
    spread = []
    for _ in range(mc):
        
        # Simulate propagation process      
        new_active, A = S[:], S[:]
        while new_active:
            
            # Get edges that flow out of each newly active node
            temp = df.loc[df['source'].isin(new_active)]

            # Extract the out-neighbors of those nodes and the weights
            targets = temp['target'].tolist()

            # Determine those neighbors that become infected
            success  = np.random.uniform(0,1,len(targets)) < p
            new_ones = list(np.extract(success, targets))
            
            # Create a list of nodes that weren't previously activated
            new_active = list(set(new_ones) - set(A))
            
            # Add newly activated nodes to the set of activated nodes
            A += new_active
            
        spread.append(len(A))
        
    return(mean(spread))

def celf(gr,k,p=0.5,mc=1000):   
    """
    Input:  dataframe representation of network, set of seed nodes, propagation probability
            and number of Monte-Carlo simulations
    Output: optimal seed set and time for each iteration
    """
      
    # --------------------
    # Find the first node with greedy algorithm
    # --------------------
    
    # Compute marginal gain for each node
    marg_gain, start_time, candidates = [], time.time(), list(set(gr['source'].tolist()))
    for node in candidates:
        marg_gain.append(IC(gr,[node],p=p,mc=mc))

    # Create the sorted list of nodes and their marginal gain 
    Q = sorted(zip(candidates,marg_gain), key = lambda x: x[1],reverse=True)

    # Select the first node and remove from candidate list
    S, spread, Q = [Q[0][0]], Q[0][1], Q[1:]
    timelapse = [time.time()-start_time]
    
    # --------------------
    # Find the next k-1 nodes using the CELF list-sorting procedure
    # --------------------
    
    for _ in range(k-1):    

        check = False
        
        while not check:
            
            # Recalculate spread of top node
            current = Q[0][0]
            
            # Evaluate the spread function and store the marginal gain in the list
            Q[0] = (current,IC(gr,S+[current],p=p,mc=mc) - spread)

            # Re-sort the list
            Q = sorted(Q, key = lambda x: x[1], reverse=True)

            # Check if previous top node stayed on top after the sort
            check = (Q[0][0] == current)

        # Select the next node
        S.append(Q[0][0])
        spread = Q[0][1]
        timelapse.append(time.time() - start_time)
        
        # Remove the selected node from the list
        Q = Q[1:]
    
    return(sorted(S),timelapse)

Unlike our comparison of the Greedy and CELF algorithms in a previous post, and unlike the results from our toy example above, there is no theoretical guarantee that the CELF and RIS approaches should yield identical results for a finite number of Monte Carlo simulations. The only guarantee is that both algorithms yield a seed set that achieves a spread within $(1-1/e-\epsilon)$ of the optimal achievable spread with a given probability, but those sets may be different.

It is worth taking the time here to discuss the difference between the mc parameter in each method, and how they relate to the error term $\epsilon$ in the approximation. In CELF, mc specifies how many different propagation cascades are run to calculate the mean spread of each node under consideration. In RIS, however, mc instead dictates the size of the collection of RRS sets R. Setting the same value of mc for each method does not necessarily, and usually will not, make for a fair comparison. Ideally, we want to set the respective values of mc to achieve the same level of accuracy $\epsilon$.

Unfortunately, this is not a trivial exercise. Tang et al. (2014) (Equation 10) showed that the RIS algorithm achieves the desired approximation with probability $1-n^{-1}$ if mc is set greater than

$$(8+2\epsilon)\cdot n\cdot \frac{\log n+\log {n\choose k}+\log 2}{\epsilon^2\cdot OPT}$$

where $OPT$ is the maximum achievable spread. Of course, $OPT$ is generally unknown so a lower bound is generally used to ensure a given $\epsilon$ is achieved. Much of the recent literature has been dedicated to making the estimation of the lower bound less conservative and faster to compute (see Tang et al. (2015), Nguyen et al. (2016), and Huang et al. (2017) among others). The equivalent expression for the CELF algorithm is

$$ (8k^2+2k\epsilon)\cdot n\cdot \frac{2\log n+\log k}{\epsilon^2\cdot OPT} $$

For our example below, in which $n=100$ and $k=10$ the expressions actually imply that mc for the CELF algorithm needs to be higher than the mc for the RIS algorithm. Because we are only interested in ensuring that the RIS algorithm achieves at least the same theoretical accuracy as the CELF algorithm, we'll nevertheless set RIS mc parameter larger than its CELF counterpart.

The graph below shows a speed comparison of the two algorithms and the relationship between computation time and the size of the seed set. The most obvious point is that the RIS algorithm is much faster (about an order of magnitude faster) despite the fact that we used a much larger mc value, which implies that it is also more accurate. The other feature of the plot is that the CELF computation time grows somewhat proportionately with the size of the seed set (although not quite, see the previous post to understand why), whereas the RIS algorithm time stays pretty much constant with respect to the size of the seed set. This is because the CELF algorithm must compute the marginal gain for a given potential seed node in each round of the iteration procedure. In contrast, RIS performs all simulations "up front" so that the only cost to expanding the seed set is the "search cost" of finding the node that appears most frequently in a list, which is a relatively cheap computational exercise.

Conclusion¶

We implemented both the CELF and RIS algorithms as simple Python functions and showed the following:

  • Both algorithms result in similar seed sets and influence spread.
  • The RIS algorithm runs a lot faster and scales much better with network size and seed set size.
